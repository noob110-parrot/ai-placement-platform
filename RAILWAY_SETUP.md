# Railway Deployment - Step by Step

## You are here: Signed into Railway ✅

Now let's deploy your entire backend stack.

---

## Step 1: Create a New Project

1. On Railway dashboard, click **"New Project"** (top right)
2. Select **"Deploy from GitHub repo"**
3. You'll be asked to authorize GitHub - click through if needed
4. Find your repository: `noob110-parrot/ai-placement-platform`
5. Click to select it

---

## Step 2: Railway Auto-Configuration

Railway will scan your repo and offer to add services.

You should see options for:
- ✅ **Python** (detected from backend/requirements.txt)
- PostgreSQL (you'll add manually)
- Redis (you'll add manually)

**Accept the Python service** - it will create a "backend" service.

---

## Step 3: Add Database Services

### Add PostgreSQL

1. In your project, click **"Add"** (+ button)
2. Select **"Database"** → **"PostgreSQL"**
3. Railway will create it automatically
4. It will auto-generate: `DATABASE_URL` environment variable

### Add Redis

1. Click **"Add"** again
2. Select **"Database"** → **"Redis"**
3. Railway creates it automatically
4. It will auto-generate: `REDIS_URL` environment variable

---

## Step 4: Set Environment Variables

Now add all the config from your `.env` file.

### In Railway Dashboard:

1. Click on the **"backend"** service
2. Go to **Variables** tab
3. Add each variable:

```
SECRET_KEY=your-super-secret-key-change-in-production-min-32-chars
ALGORITHM=HS256
ACCESS_TOKEN_EXPIRE_MINUTES=30
OPENAI_API_KEY=sk-...
OPENAI_MODEL=gpt-4o-mini
WHATSAPP_API_TOKEN=your-whatsapp-api-token
WHATSAPP_PHONE_NUMBER_ID=your-phone-number-id
WHATSAPP_BUSINESS_ACCOUNT_ID=your-business-account-id
WHATSAPP_VERIFY_TOKEN=your-webhook-verify-token
GOOGLE_CLIENT_ID=your-google-client-id.apps.googleusercontent.com
GOOGLE_CLIENT_SECRET=your-google-client-secret
GOOGLE_REDIRECT_URI=https://your-frontend-vercel-url/api/auth/callback/google
GOOGLE_CALENDAR_SCOPES=https://www.googleapis.com/auth/calendar
N8N_WEBHOOK_URL=http://localhost:5678
N8N_API_KEY=your-n8n-api-key
N8N_BASIC_AUTH_USER=admin
N8N_BASIC_AUTH_PASSWORD=placement123
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USER=your-placement-cell@gmail.com
SMTP_PASSWORD=your-app-password
EMAIL_FROM=AI Placement Platform <placement@university.edu>
ENABLE_WHATSAPP_NOTIFICATIONS=true
ENABLE_EMAIL_NOTIFICATIONS=true
ENABLE_CALENDAR_SYNC=true
ENABLE_AI_SUMMARIZER=true
```

**Note:** `DATABASE_URL` and `REDIS_URL` are auto-generated by Railway - don't manually add them.

---

## Step 5: Configure the Start Command

Railway should auto-detect from Procfile, but to be sure:

1. Click on **backend** service
2. Go to **Settings** tab
3. Find "Build Command" or "Start Command"
4. Verify it shows: `uvicorn app.main:app --host 0.0.0.0 --port $PORT`

If not, set it manually.

---

## Step 6: Deploy

1. Click the **"Deploy"** button
2. Railway will:
   - Build the Docker image
   - Install dependencies from requirements.txt
   - Start the uvicorn server
   - Generate a public URL

⏳ Wait 3-5 minutes for the build and deployment...

---

## Step 7: Get Your Backend Public URL

Once deployed:

1. Go to your Railway project
2. Click on the **"backend"** service
3. Find the **"Public URL"** section (blue button on top-right)
4. Copy the URL - looks like: `https://backend-production.up.railway.app`

---

## Step 8: Update Vercel Frontend

Now update the frontend to use your Railway backend:

1. Go to https://vercel.com/dashboard
2. Click on your **"frontend"** project
3. Go to **Settings** → **Environment Variables**
4. Find or create: `NEXT_PUBLIC_API_URL`
5. Set its value to your Railway backend URL:
   ```
   NEXT_PUBLIC_API_URL=https://backend-production.up.railway.app
   ```
6. Save and redeploy:
   - Go to **Deployments** tab
   - Click the 3-dot menu on latest deployment
   - Select **"Redeploy"**

⏳ Wait 2-3 minutes for Vercel to rebuild...

---

```

```

## Step 10: Optional - Add PgAdmin and n8n

### PgAdmin (Database Manager UI)

1. In Railway project, click **"Add"**
2. Select **"Container"**
3. Image: `dpage/pgadmin4:latest`
4. Environment variables:
   ```
   PGADMIN_DEFAULT_EMAIL=admin@placement.edu
   PGADMIN_DEFAULT_PASSWORD=admin
   ```
5. Deploy
6. Access via public URL with admin/admin

### n8n (Workflow Automation)

1. In Railway project, click **"Add"**
2. Select **"Container"**
3. Image: `n8nio/n8n:latest`
4. Environment variables:
   ```
   N8N_BASIC_AUTH_ACTIVE=true
   N8N_BASIC_AUTH_USER=admin
   N8N_BASIC_AUTH_PASSWORD=placement123
   N8N_PROTOCOL=https
   ```
5. Deploy
6. Access via public URL with admin/placement123

## Troubleshooting

### Backend won't start
- Check Railway logs: Click backend service → "Logs" tab
- Look for errors in the build output
- Common issues:
  - Missing environment variables
  - DATABASE_URL/REDIS_URL not auto-generated

### Frontend can't reach backend
- Verify `NEXT_PUBLIC_API_URL` is set in Vercel
- Check browser console (F12) for CORS or network errors
- Ensure Railway backend URL is correct and public

### Database connection fails
- Railway PostgreSQL auto-creates - just wait
- Use PgAdmin to verify database exists
- Check `DATABASE_URL` in Railway Variables tab
